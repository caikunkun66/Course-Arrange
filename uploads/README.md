# 文件上传目录说明

## 目录用途
此目录用于存储用户上传的文件，包括学生头像等。

## 目录结构
```
uploads/
└── avatars/              # 学生头像存储目录
    └── 20251030/         # 按日期分类（yyyyMMdd）
        └── *.jpg/png     # 头像文件
```

## 注意事项
1. 此目录中的文件不会被提交到 Git 仓库（已添加到 .gitignore）
2. 首次运行项目时，会自动创建所需的子目录
3. 上传的文件通过 `/uploads/**` 路径访问
4. 建议定期备份重要文件

## 访问方式
上传的文件可以通过以下方式访问：
```
http://localhost:8008/uploads/avatars/日期/文件名
```

## 生产环境
部署到生产环境时，建议：
- 使用独立的文件服务器或对象存储（如阿里云 OSS）
- 配置 Nginx 处理静态文件访问
- 实施文件访问权限控制
- 定期清理无用文件
